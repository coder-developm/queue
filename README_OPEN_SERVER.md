# Электронная очередь — фронт + PHP API (OpenServer)

## Что внутри
- Статический фронт (твоя вёрстка): `/index.html`, `/poster`, `/status`, `/a`, `/create`
- PHP API: `/api/index.php`
- MySQL схема: `/sql/schema.sql`
- Rewrite правила: `/.htaccess`
- Общая мультиязычность RU/EN: `/js/i18n.js` (используется страницей очереди и табло)

## Быстрый тест на OpenServer (Windows)

### 1) Скопируй проект
1. Распакуй архив в: `OpenServer/domains/queue.local/`
2. В OpenServer выбери:
   - Сервер: **Apache**
   - PHP: **8.x** (можно 7.4+, но лучше 8)
   - MySQL: **MySQL 5.7/8.0**

### 2) Создай базу и таблицы
1. Открой phpMyAdmin (в OpenServer)
2. Создай БД, например: `queue`
3. Выполни SQL из файла: `sql/schema.sql`

> Если ты уже разворачивал предыдущую версию проекта, проще всего **переимпортировать БД с нуля**.
> Если не хочется — добавь изменения вручную:
> - `admin_users`: поля `permissions_json`, `comment`
> - `create_tokens`: поле `created_by`
> - `queues`: поле `input_mask`
> - `site_branding`: поле `theme_mode`, `notify_sound_url` + строка `page_key='global'`
> - `create_tokens.created_by` должен ссылаться на `admin_users.id` (FOREIGN KEY)
> Полная актуальная схема — в `sql/schema.sql`.

### 3) Настрой .env
1. Скопируй `.env.example` → `.env`
2. Заполни:
   - `DB_DSN=mysql:host=127.0.0.1;dbname=queue;charset=utf8mb4`
   - `DB_USER=root`
   - `DB_PASS=` (в OpenServer часто пустой пароль)

> Мини‑админка больше не использует логин/пароль из `.env` — они хранятся в БД.

### 4) Включи mod_rewrite
Обычно в OpenServer уже включено.
Важно: файл `.htaccess` должен лежать в корне сайта рядом с `index.html`.

### 5) Проверь API
Открой:
- `http://queue.local/api/health` → должно вернуть `{ "ok": true }`

## Как пользоваться
### Создание очереди
Страница создания теперь защищена.

1) Открой мини‑админку:
- `http://queue.local/admin/`

2) Если это первый запуск и пользователей ещё нет, появится **форма первичной настройки** — создай первый аккаунт.

3) Нажми **"Создать ссылку"** → получишь URL вида:
- `http://queue.local/create/<TOKEN>`

4) Перейди по этой ссылке и создай очередь.
После создания откроется страница `/create/success` с подробной информацией и кнопкой копирования.

### Очередь (для клиента)
- `http://queue.local/{publicId}`

### Табло
- `http://queue.local/status/{publicId}`

Тексты табло и **озвучка** зависят от настройки **"Язык экрана состояния"** (в админке → ⚙ → Настройки очереди).

### Админка
- `http://queue.local/a/{publicId}/{adminToken}`

## Поведение админки (как в примере)
- Кнопка **"Касса"** открывает модалку **"Рабочее место"** и позволяет выбрать кассу.
- **Пауза**: при первом нажатии ставит кассу на паузу и показывает красный алерт. После первого нажатия появляется кнопка **"скрыть кассу"** (зачёркнутый глаз).
- **Скрыть кассу**: меняет текст алерта ("Касса скрыта") и сама кнопка скрывается (как в примере).
- **Пригласить вне очереди** и **Удаление из очереди** открывают нижний шит со списком ожидающих. Выбираешь посетителя → действие.
- **Вернуть в очередь** активен только когда на выбранной кассе есть приглашённый номер.

## Настройки очереди (теперь сохраняются в БД)
В админке открой **⚙ → Настройки очереди** и нажми ✓ для сохранения.

Сохраняются:
- **Самостоятельная регистрация** (если выключить — посетители не смогут зайти в очередь по ссылке)
- **Просить ввод данных посетителя** + **Подсказка**
- **Вместимость очереди** (ограничение на количество ожидающих)
- **Язык экрана состояния очереди** (Auto / Русский / English) — влияет на тексты табло и язык озвучки.
- **Брендинг: основной/доп. цвет, логотипы (загрузка файлов) и тема (светлая/тёмная/авто) — меняется в мини‑админке во вкладке **Брендинг** и применяется ко всем очередям/страницам.
  Дополнительно там же можно загрузить **звук уведомления** (WAV/MP3/OGG) — он используется на табло перед озвучкой и у клиента при приглашении.
  Важно: тёмная тема применяется **только** к странице табло (`/status/...`). Остальные страницы всегда светлые.

## Услуги по кассам (рабочее место)
В модалке **"Рабочее место"** можно выбрать:
- номер кассы
- какие услуги эта касса обслуживает (если услуги добавлены при создании очереди)

Если у кассы выбраны услуги, то кнопка **"Следующий"** будет звать только тех посетителей, у кого услуга входит в список.
Если касса одна — неподходящие услуги подсвечиваются красным.

## Примечания
- Озвучка на табло использует `SpeechSynthesis` браузера и переключатель звука на странице.
- В режиме языка **Auto** язык определяется по браузеру на устройстве табло.
- Если хочешь 3+ кассы, фронт (табло/админка) нужно расширить — backend это поддерживает.


## Частые проблемы

### 1) Домен не открывается (DNS)
Если браузер пишет "не удалось найти IP-адрес", проверь:
- папка проекта лежит в `OpenServer/domains/<имя_домена>/`
- открываешь именно `http://<имя_домена>/`
- OpenServer запущен (лучше от администратора)
- в `C:\Windows\System32\drivers\etc\hosts` есть строка:
  `127.0.0.1 <имя_домена>`
После изменения hosts можно выполнить `ipconfig /flushdns`.

### 2) Ошибка MySQL Access denied
В OpenServer чаще всего:
- `DB_USER=root`
- `DB_PASS=` (пустой пароль)

Проверь в `.env`.

### 3) Стили/скрипты не грузятся
Убедись, что `.htaccess` находится в корне рядом с `index.html`, и выбран **Apache** (не nginx).


## Важно про "красивые" ссылки
В этом проекте страницы открываются по URL вида `/status/1234567`, `/a/1234567/TOKEN` и т.д. Поэтому все CSS/JS подключаются **абсолютными путями** (например `/status/js/script.js`). Если у вас в DevTools ошибка `Unexpected token '<'` — почти всегда это значит, что браузеру вместо JS вернулся HTML из-за неверного пути.


## Мини‑админка: безопасная аутентификация

Логин/пароль хранятся в БД в таблице `admin_users` (пароль — как `password_hash`).
При первом открытии `/admin/` (если пользователей ещё нет) появится форма первичной настройки — создайте первый аккаунт.

Защита от перебора:
- максимум 5 неудачных попыток за 10 минут с одного IP
- блокировка на 15 минут после превышения лимита

Если нужно создать пользователя вручную (через phpMyAdmin), вставляйте `password_hash`, полученный в PHP:
```php
<?php echo password_hash('YOUR_PASSWORD', PASSWORD_DEFAULT);
```

### "Запомнить меня", сессии и смена пароля
- На форме входа есть чекбокс **"Запомнить меня"**: создаётся длинная cookie‑сессия (по умолчанию 30 дней).
- Вкладка **"Безопасность"** показывает **короткие** и **долгие** сессии, можно завершать отдельные.
- Кнопка **"Выйти со всех устройств"** завершает ВСЕ сессии, включая текущую (перебросит на страницу входа).
- Смена пароля делается во вкладке **"Безопасность"**.

### Несколько админов и доступ к разделам
Первый созданный пользователь имеет роль **owner**.

Вкладка **"Пользователи"** доступна только owner и позволяет:
- создавать новых админов
- указывать комментарий
- включать/выключать доступ к разделам мини‑админки чекбоксами:
  - **Ссылки**
  - **Брендинг**
  - **Пользователи**

Особенности:
- пользователя с ролью **owner** удалить нельзя (кнопка удаления не показывается)
- owner всегда имеет полный доступ независимо от чекбоксов

```bash
php tools/generate_vapid_keys.php mailto:admin@example.com
```

Скопируйте `VAPID_PUBLIC_KEY` и `VAPID_PRIVATE_KEY` в `.env`.
